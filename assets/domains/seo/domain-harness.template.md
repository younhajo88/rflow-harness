# SEO Domain Harness

## Domain Mission

Build public web projects that can be crawled, indexed, understood, and later observed through trustworthy search data.

## Domain Roles

- Topic Researcher: map search intent and content opportunities.
- Site Architect: define crawlable route and content structure.
- Technical SEO Reviewer: verify metadata, canonical URLs, robots, sitemap, structured data, HTTP behavior, and renderability.
- Growth Analyst: interpret Search Console or other observed data after launch.

## Guardrails

- Do not claim Google ranking or indexing is guaranteed.
- Do not invent Search Console, analytics, or keyword exposure data.
- Separate crawl/index blockers from ranking-support recommendations.
- Verify public HTML, status codes, metadata, canonical URLs, sitemap, robots, structured data, language, and internal links before release readiness.
- Treat Search Console actions as manual unless authenticated access is explicitly available.

## Release Review Sources

When current SEO release readiness matters, use official current documentation:

- Google Search Essentials
- Google SEO Starter Guide
- Google JavaScript SEO basics
- Google structured data policies
- Google favicon and appearance guidance
- Next.js App Router metadata, sitemap, and robots documentation

## Phase Additions

SEO phase plans should include:

- URL matrix
- search intent target
- metadata expectations
- canonical strategy
- sitemap and robots expectations
- structured data expectations
- local fetch checks
- production URL checks when deployed
- Search Console manual actions when relevant
